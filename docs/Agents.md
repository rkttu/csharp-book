# C# 프로그래밍 실습 책 작성 가이드

이 문서는 "Visual Studio Code와 .NET 10 File-based App을 활용한 C# 프로그래밍 학습서"의 일관성을 유지하기 위한 가이드입니다.

## 책의 핵심 철학

### 1. File-based App 중심 접근
- 복잡한 프로젝트 구조 없이 단일 `.cs` 파일로 시작
- `.csproj` 파일이나 솔루션 파일 없이 간단하게 학습
- `dotnet run`으로 즉시 실행 가능한 예제
- 필요에 따라 점진적으로 프로젝트 구조 도입

### 2. Visual Studio Code 활용
- Visual Studio Code를 주 개발 도구로 사용
- C# 확장 프로그램 활용
- 터미널 기반의 .NET CLI 명령어 사용
- 디버깅, IntelliSense 등 VS Code 기능 최대 활용

### 3. .NET 10 기준
- .NET 10의 최신 기능 활용
- Modern C# 기능 (C# 10+) 적극 사용
- File-scoped namespaces, global using 등 최신 문법 활용

## 작성 지침

### 코드 예제 작성

1. **간결성**: 모든 예제는 가능한 한 간결하고 이해하기 쉽게 작성
2. **실행 가능성**: 예제 코드는 복사-붙여넣기로 즉시 실행 가능해야 함
3. **점진적 복잡도**: 간단한 예제에서 시작하여 점진적으로 복잡도 증가

```csharp
// 좋은 예제 (File-based App 스타일)
Console.WriteLine("Hello, C#!");

var numbers = new[] { 1, 2, 3, 4, 5 };
var sum = numbers.Sum();
Console.WriteLine($"합계: {sum}");
```

### 설명 스타일

1. **한글 중심**: 모든 설명은 한글로 작성 (기술 용어는 영문 병기 가능)
2. **초보자 친화적**: C# 입문자도 이해할 수 있는 설명
3. **실용적 접근**: 이론보다는 실용적이고 실습 가능한 내용 중심
4. **단계별 진행**: 각 개념을 단계별로 나누어 설명

### 장(Chapter) 구성

각 장은 다음과 같은 구조로 작성:

1. **도입부**: 해당 장에서 배울 내용 개요
2. **개념 설명**: 핵심 개념을 간단명료하게 설명
3. **코드 예제**: 실행 가능한 예제 코드와 설명
4. **실습 문제**: 학습자가 직접 해볼 수 있는 연습 문제
5. **정리**: 핵심 내용 요약

### File-based App에서 프로젝트로의 전환

책의 후반부로 갈수록:
- File-based App에서 시작
- 필요에 따라 클래스 파일 분리
- 프로젝트 구조 도입 (`.csproj` 사용)
- NuGet 패키지 추가
- 단위 테스트 프로젝트 추가

## 용어 사용 가이드

### 일관된 용어 사용

| 한글 용어 | 영문 용어 | 사용 예 |
|----------|----------|---------|
| 메서드 | Method | "메서드(Method)를 정의합니다" |
| 속성 | Property | "속성(Property)을 통해 접근합니다" |
| 필드 | Field | "private 필드(Field)를 선언합니다" |
| 클래스 | Class | "클래스(Class)를 정의합니다" |
| 인터페이스 | Interface | "인터페이스(Interface)를 구현합니다" |
| 람다 식 | Lambda Expression | "람다 식(Lambda Expression)을 사용합니다" |
| 비동기 | Async/Asynchronous | "비동기(Async) 메서드를 작성합니다" |
| 컬렉션 | Collection | "List 컬렉션(Collection)을 사용합니다" |

### 코드 주석

```csharp
// 한글로 간결한 주석 작성
var name = "홍길동";  // 변수 초기화

// 복잡한 로직은 단계별로 설명
// 1. 데이터 필터링
var filtered = data.Where(x => x.IsActive);

// 2. 정렬
var sorted = filtered.OrderBy(x => x.Name);

// 3. 결과 반환
return sorted.ToList();
```

## 실습 프로젝트 가이드

### 프로젝트 선정 기준

1. **실용성**: 실생활에서 사용할 수 있는 프로젝트
2. **적절한 난이도**: 해당 시점까지 배운 내용으로 구현 가능
3. **확장 가능성**: 기본 기능 구현 후 추가 기능 구현 가능
4. **학습 효과**: 여러 개념을 통합적으로 활용

### 프로젝트 진행 방식

1. **요구사항 분석**: 만들고자 하는 프로그램의 기능 정의
2. **설계**: 필요한 클래스와 메서드 설계
3. **구현**: 단계별로 기능 구현
4. **테스트**: 각 기능이 제대로 동작하는지 확인
5. **개선**: 코드 리팩토링 및 기능 추가

### 클래스 정의 예제

```csharp
// [장 번호].[절 번호] [제목]

// 클래스 정의
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    
    public void Introduce()
    {
        Console.WriteLine($"안녕하세요, {Name}입니다. {Age}살입니다.");
    }
}

// 사용 예제
var person = new Person
{
    Name = "김철수",
    Age = 25
};
person.Introduce();

// 출력: 안녕하세요, 김철수입니다. 25살입니다.
```
