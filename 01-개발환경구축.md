C# 프로그래밍을 시작하기 위해서는 먼저 개발 환경을 준비해야 합니다. 이 장에서는 Visual Studio Code와 .NET 10 SDK를 설치하고, 첫 번째 프로그램을 작성하여 실행하는 방법을 배웁니다.

## 이 장에서 배울 내용
- Visual Studio Code 설치 방법
- .NET 10 SDK 설치 및 확인
- C# 개발을 위한 확장 프로그램 설치

## 1.1 Visual Studio Code 설치

Visual Studio Code(VS Code)는 Microsoft에서 개발한 무료 코드 에디터로, 가볍고 강력한 기능을 제공합니다. 다양한 프로그래밍 언어를 지원하며, 특히 C# 개발에 최적화된 환경을 제공합니다.

### Visual Studio Code의 장점

1. **무료 오픈소스**: 누구나 무료로 사용할 수 있습니다
2. **크로스 플랫폼**: Windows, macOS, Linux에서 모두 동작합니다
3. **가벼움**: 빠른 시작과 실행 속도를 자랑합니다
4. **풍부한 확장 기능**: 필요한 기능을 확장 프로그램으로 추가할 수 있습니다
5. **IntelliSense**: 코드 자동 완성과 힌트 제공
6. **통합 터미널**: 에디터 내에서 명령어를 실행할 수 있습니다

### 설치 방법

#### Windows

1. 웹 브라우저에서 https://code.visualstudio.com/ 접속
2. "Download for Windows" 버튼 클릭
3. 다운로드한 설치 파일 실행
4. 설치 마법사 따라 진행:
   - "I accept the agreement" 선택
   - 설치 위치 선택 (기본값 권장)
   - 추가 작업 선택 시 다음 항목 체크:
     - ✅ "Add to PATH" (환경 변수에 추가)
     - ✅ "Create a desktop icon" (바탕화면 아이콘 생성)
     - ✅ "Register Code as an editor for supported file types" (기본 에디터로 등록)
5. "Install" 버튼 클릭하여 설치 완료

#### macOS

1. 웹 브라우저에서 https://code.visualstudio.com/ 접속
2. "Download for Mac" 버튼 클릭
3. 다운로드한 .zip 파일을 압축 해제
4. Visual Studio Code.app을 응용 프로그램 폴더로 이동
5. Launchpad에서 Visual Studio Code 실행

#### Linux (Ubuntu/Debian)

```bash
# 터미널에서 다음 명령어 실행
sudo apt update
sudo apt install code
```

또는 https://code.visualstudio.com/docs/setup/linux 에서 배포판별 설치 방법을 확인할 수 있습니다.

### 설치 확인

Visual Studio Code를 실행하여 정상적으로 설치되었는지 확인합니다.

1. Visual Studio Code 실행
2. 시작 화면이 표시되면 설치 성공
3. 단축키 확인:
   - Windows/Linux: `Ctrl + Shift + P` (명령 팔레트)
   - macOS: `Cmd + Shift + P` (명령 팔레트)

## 1.2 .NET 10 SDK 설치 및 확인

.NET 10 SDK는 C# 프로그램을 개발하고 실행하기 위해 필요한 도구 모음입니다. SDK에는 컴파일러, 런타임, 라이브러리 등이 포함되어 있습니다.

### .NET SDK vs .NET Runtime

- **.NET SDK**: 개발에 필요한 모든 도구 포함 (개발자용)
- **.NET Runtime**: 프로그램 실행만 가능 (사용자용)

개발을 위해서는 반드시 **.NET SDK**를 설치해야 합니다.

### 설치 방법

#### Windows

1. 웹 브라우저에서 https://dotnet.microsoft.com/download 접속
2. ".NET 10" 또는 최신 버전의 SDK 선택
3. "Download .NET SDK x64" 버튼 클릭
4. 다운로드한 설치 파일 실행
5. 설치 마법사를 따라 진행
6. 설치 완료 후 컴퓨터 재시작 (권장)

#### macOS

```bash
# Homebrew를 사용한 설치 (권장)
brew install dotnet-sdk

# 또는 공식 웹사이트에서 설치 프로그램 다운로드
# https://dotnet.microsoft.com/download
```

#### Linux (Ubuntu/Debian)

```bash
# 터미널에서 다음 명령어 실행
wget https://packages.microsoft.com/config/ubuntu/22.04/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb

sudo apt update
sudo apt install -y dotnet-sdk-10.0
```

### 설치 확인

터미널 또는 명령 프롬프트를 열어 다음 명령어를 실행합니다:

```bash
dotnet --version
```

**출력 예시:**
```
10.0.100
```

버전 번호가 표시되면 설치가 성공적으로 완료된 것입니다.

### 추가 정보 확인

.NET SDK에 대한 더 자세한 정보를 확인하려면:

```bash
dotnet --info
```

**출력 예시:**
```
.NET SDK:
 Version:           10.0.100
 Commit:            abc1234def

Runtime Environment:
 OS Name:           Windows
 OS Version:        10.0.22631
 OS Platform:       Windows
 RID:               win-x64
 Base Path:         C:\Program Files\dotnet\sdk\10.0.100\

Host:
  Version:         10.0.0
  Architecture:    x64
  ...
```

이 명령어는 설치된 .NET의 버전, 운영체제 정보, 설치 경로 등을 보여줍니다.

### 도움말 확인

.NET CLI의 사용 가능한 명령어 목록을 보려면:

```bash
dotnet --help
```

## 1.3 C# 확장 프로그램 설치

Visual Studio Code에서 C# 개발을 편리하게 하기 위해서는 C# 확장 프로그램을 설치해야 합니다. 이 확장 프로그램은 코드 자동 완성, 디버깅, 구문 강조 등의 기능을 제공합니다.

### C# Dev Kit 설치

Microsoft에서 제공하는 공식 C# Dev Kit는 C# 개발에 필요한 모든 기능을 포함하고 있습니다.

#### 설치 방법

1. Visual Studio Code 실행
2. 좌측 사이드바에서 확장(Extensions) 아이콘 클릭 (단축키: `Ctrl+Shift+X` 또는 `Cmd+Shift+X`)
3. 검색창에 "C# Dev Kit" 입력
4. Microsoft에서 제공하는 "C# Dev Kit" 찾기
5. "Install" 버튼 클릭
6. 설치 완료 후 "Reload" 버튼 클릭 (필요한 경우)

#### C# Dev Kit의 주요 기능

- **IntelliSense**: 코드 자동 완성
- **구문 강조**: 코드 가독성 향상
- **디버깅**: 중단점 설정 및 변수 검사
- **코드 탐색**: 정의로 이동, 참조 찾기
- **리팩토링**: 코드 개선 도구
- **테스트 지원**: 단위 테스트 실행

### 추가 유용한 확장 프로그램

필수는 아니지만, 다음 확장 프로그램들도 개발에 도움이 됩니다:

1. **C#** (기본 C# 언어 지원)
   - 검색: "C#"
   - 제공: Microsoft

2. **NuGet Package Manager**
   - 검색: "NuGet Package Manager"
   - NuGet 패키지 관리 지원

3. **Better Comments**
   - 검색: "Better Comments"
   - 주석을 색상으로 구분하여 가독성 향상

4. **Error Lens**
   - 검색: "Error Lens"
   - 오류를 코드 옆에 바로 표시

### 확장 프로그램 설치 확인

1. Visual Studio Code에서 `.cs` 파일 생성
2. 다음 코드 입력:

```csharp
Console.
```

3. `Console.` 뒤에 점을 입력하면 자동 완성 목록이 표시되면 성공

## 1.4 첫 번째 File-based App 만들기

.NET 10에서는 **File-based App** 방식으로 간단하게 C# 프로그램을 작성할 수 있습니다. 프로젝트 파일(`.csproj`)이나 복잡한 구조 없이 `.cs` 파일 하나만으로 프로그램을 만들 수 있습니다.

### File-based App이란?

전통적인 C# 프로젝트는 여러 파일과 설정이 필요했지만, File-based App은:
- ✅ 단일 `.cs` 파일만으로 실행 가능
- ✅ `Main` 메서드 없이 최상위 문(Top-level statements) 사용
- ✅ 네임스페이스 선언 불필요
- ✅ 빠른 프로토타이핑과 학습에 적합

### 첫 번째 프로그램 작성

#### 1단계: 작업 폴더 만들기

먼저 C# 프로젝트를 저장할 폴더를 만듭니다.

```bash
# Windows
mkdir C:\CSharp-Practice
cd C:\CSharp-Practice

# macOS/Linux
mkdir ~/CSharp-Practice
cd ~/CSharp-Practice
```

#### 2단계: Visual Studio Code로 폴더 열기

1. Visual Studio Code 실행
2. "File" > "Open Folder..." 선택
3. 방금 만든 `CSharp-Practice` 폴더 선택
4. "Select Folder" 버튼 클릭

#### 3단계: 새 파일 생성

1. 좌측 탐색기에서 "New File" 아이콘 클릭
2. 파일 이름을 `Program.cs`로 입력
3. Enter 키를 눌러 파일 생성

#### 4단계: Hello World 프로그램 작성

`Program.cs` 파일에 다음 코드를 입력합니다:

```csharp
// 1.4 첫 번째 File-based App
// 설명: 콘솔에 메시지를 출력하는 간단한 프로그램

Console.WriteLine("Hello, C#!");
Console.WriteLine("안녕하세요, C# 프로그래밍 세계에 오신 것을 환영합니다!");

// 출력:
// Hello, C#!
// 안녕하세요, C# 프로그래밍 세계에 오신 것을 환영합니다!
```

#### 코드 설명

- `Console.WriteLine()`: 콘솔에 텍스트를 출력하고 줄바꿈하는 메서드
- 문자열은 큰따옴표(`"`)로 감싸서 표현
- 각 명령문은 세미콜론(`;`)으로 끝남
- `//`로 시작하는 줄은 주석으로, 프로그램 실행에 영향을 주지 않음

### 다양한 예제

#### 예제 1: 여러 줄 출력

```csharp
// 여러 줄의 메시지 출력
Console.WriteLine("첫 번째 줄");
Console.WriteLine("두 번째 줄");
Console.WriteLine("세 번째 줄");

// 출력:
// 첫 번째 줄
// 두 번째 줄
// 세 번째 줄
```

#### 예제 2: 변수 사용

```csharp
// 변수 선언과 사용
var name = "김철수";
var age = 25;

Console.WriteLine("이름: " + name);
Console.WriteLine("나이: " + age);

// 출력:
// 이름: 김철수
// 나이: 25
```

#### 예제 3: 문자열 보간 (String Interpolation)

```csharp
// 문자열 보간을 사용한 출력
var name = "이영희";
var age = 30;
var city = "서울";

Console.WriteLine($"안녕하세요! 저는 {city}에 사는 {age}살 {name}입니다.");

// 출력:
// 안녕하세요! 저는 서울에 사는 30살 이영희입니다.
```

문자열 보간은 `$` 기호를 문자열 앞에 붙이고, 중괄호 `{}`로 변수를 감싸서 사용합니다. 이 방법이 문자열 연결(`+`)보다 읽기 쉽고 편리합니다.

#### 예제 4: 간단한 계산

```csharp
// 간단한 산술 연산
var num1 = 10;
var num2 = 20;
var sum = num1 + num2;

Console.WriteLine($"{num1} + {num2} = {sum}");

// 출력:
// 10 + 20 = 30
```

## 1.5 실행 및 디버깅

작성한 C# 프로그램을 실행하고 문제를 해결하는 방법을 배웁니다.

### 프로그램 실행하기

#### 방법 1: 터미널에서 실행

Visual Studio Code의 통합 터미널을 사용하는 방법입니다.

1. Visual Studio Code에서 터미널 열기:
   - 메뉴: "Terminal" > "New Terminal"
   - 단축키: ``Ctrl + ` `` (백틱, 물결표 아래 키)

2. 터미널에 다음 명령어 입력:

```bash
dotnet run Program.cs
```

또는 파일 이름을 생략하고:

```bash
dotnet run
```

3. 프로그램이 실행되고 출력 결과가 터미널에 표시됩니다.

#### 방법 2: 코드 실행 단축키

1. `Program.cs` 파일을 열어놓은 상태에서
2. `F5` 키 또는 상단 메뉴에서 "Run" > "Start Debugging" 선택
3. 첫 실행 시 launch.json 구성 선택 화면이 나타나면 "C#" 선택
4. 프로그램이 실행되고 디버그 콘솔에 출력이 표시됩니다

### 디버깅 기초

디버깅(Debugging)은 프로그램의 오류를 찾고 수정하는 과정입니다.

#### 중단점(Breakpoint) 설정

중단점은 프로그램 실행을 특정 위치에서 일시 정지시켜 상태를 확인할 수 있게 합니다.

1. 디버깅하고 싶은 코드 줄의 줄 번호 왼쪽을 클릭
2. 빨간 점이 표시되면 중단점 설정 완료
3. `F5`를 눌러 디버깅 시작
4. 프로그램이 중단점에서 멈춤

#### 디버깅 예제

`Program.cs` 파일을 다음과 같이 수정합니다:

```csharp
// 디버깅 연습
var a = 5;
var b = 10;
var result = a + b;  // 이 줄에 중단점 설정

Console.WriteLine($"결과: {result}");

var c = result * 2;  // 이 줄에도 중단점 설정
Console.WriteLine($"최종 결과: {c}");
```

**디버깅 단계:**

1. 3번 줄(`var result = a + b;`)에 중단점 설정
2. `F5`를 눌러 디버깅 시작
3. 프로그램이 중단점에서 멈추면 좌측 "VARIABLES" 패널에서 변수 값 확인
   - `a = 5`
   - `b = 10`
4. `F10` (Step Over)를 눌러 다음 줄로 이동
5. `result = 15`로 설정된 것을 확인
6. `F5`를 다시 눌러 다음 중단점까지 이동 또는 계속 실행

#### 디버깅 단축키

| 단축키 | 기능 | 설명 |
|--------|------|------|
| `F5` | Continue | 다음 중단점까지 실행 또는 프로그램 종료까지 실행 |
| `F10` | Step Over | 다음 줄로 이동 (함수는 건너뜀) |
| `F11` | Step Into | 다음 줄로 이동 (함수 내부로 진입) |
| `Shift+F11` | Step Out | 현재 함수에서 빠져나옴 |
| `Shift+F5` | Stop | 디버깅 중지 |
| `Ctrl+Shift+F5` | Restart | 디버깅 재시작 |

### 변수 검사

디버깅 중에 변수의 값을 확인하는 방법:

1. **VARIABLES 패널**: 좌측에 현재 스코프의 모든 변수가 표시됨
2. **마우스 호버**: 코드의 변수에 마우스를 올리면 현재 값이 표시됨
3. **WATCH 패널**: 특정 변수나 표현식을 추가하여 지속적으로 모니터링

#### Watch 표현식 추가

1. 디버깅 중 좌측 "WATCH" 패널 선택
2. "+" 버튼 클릭
3. 확인하고 싶은 변수나 표현식 입력 (예: `a + b`, `result * 2`)
4. 프로그램이 실행되면서 값이 업데이트되는 것을 확인

### 일반적인 오류와 해결 방법

#### 오류 1: 세미콜론 누락

```csharp
// 잘못된 코드
Console.WriteLine("Hello")  // 세미콜론 없음

// 올바른 코드
Console.WriteLine("Hello");  // 세미콜론 추가
```

**오류 메시지:**
```
error CS1002: ; expected
```

#### 오류 2: 따옴표 불일치

```csharp
// 잘못된 코드
Console.WriteLine("Hello');  // 따옴표 종류가 다름

// 올바른 코드
Console.WriteLine("Hello");  // 같은 종류의 따옴표 사용
```

**오류 메시지:**
```
error CS1010: Newline in constant
```

#### 오류 3: 변수 이름 오타

